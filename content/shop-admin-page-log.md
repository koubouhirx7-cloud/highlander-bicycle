# ネットショップ管理画面 実装ログ

日付: 2026-07-02

## 実装した理由

ネットショップでGoogleログインを使って購入してもらう方針になったため、注文履歴と顧客情報を後から確認できる管理者専用の入口が必要になった。

顧客管理v2へ直接すべてを自動連携する前段階として、まずは「注文データを安全に保存し、管理者が確認・補正・手動紐づけできる状態」を作る。

## 判断

- 会員ログインと管理者ログインは分ける。
- 管理画面は表示だけでなく、API側でも管理者セッションを確認する。
- 管理者パスワードはファイルに直書きせず、`.env` の `ADMIN_PASSWORD_HASH` で管理する。
- 顧客情報には、顧客管理v2への連携状態、外部ID、管理メモを持たせる。
- 注文履歴スクショやAI読み取りで補完した情報は、まず管理メモに残せる形にする。

## 今後の拡張候補

- 顧客管理v2のAPIが固まったら、管理画面からワンクリックで顧客情報を送る。
- 管理者を複数人にする場合は、パスワード1個ではなく管理者テーブルへ移行する。
- 住所・電話番号の編集履歴を残す。
- 注文ごとの対応ステータスを追加する。
